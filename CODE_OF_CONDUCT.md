# Code Of Conduct

## Our Standard

This project is an open research infrastructure effort. We expect contributors and maintainers to keep collaboration:

- respectful;
- technically honest;
- evidence-driven;
- inclusive of contributors with different backgrounds and experience levels;
- focused on the work rather than personal attacks.

## Expected Behavior

- Use clear and respectful language.
- Critique ideas, code, assumptions, and methods rather than people.
- Be explicit about uncertainty and evidence.
- Accept correction when new facts or better reasoning are presented.
- Keep disagreements professional and grounded in the project’s goals.
- Respect the difference between public discussion and private security disclosure.

## Unacceptable Behavior

- harassment, insults, or intimidation;
- discriminatory or exclusionary language;
- bad-faith technical argumentation;
- doxxing or disclosure of private information;
- publishing unpatched security issues irresponsibly;
- knowingly uploading inappropriate or rights-violating material.

## Scope

This code of conduct applies in:

- repository discussions;
- pull requests;
- issue trackers;
- review comments;
- project chat or coordination spaces tied to the repository.

## Maintainer Responsibilities

Maintainers are responsible for:

- enforcing standards consistently;
- distinguishing disagreement from misconduct;
- correcting harmful behavior proportionally;
- protecting contributors from harassment and retaliation.

## Reporting

If you experience or observe unacceptable behavior, report it privately to the maintainers through the canonical Gitea project administration path.

For security issues, use the process in [SECURITY.md](SECURITY.md) instead of public issue threads.

## Enforcement

Possible maintainer actions include:

- private warning;
- moderation of comments;
- temporary participation restrictions;
- removal from collaboration spaces;
- rejection of contributions;
- permanent ban for severe or repeated misconduct.

## Interpretation

This document is intentionally short. It is meant to support professional collaboration, not create ambiguity around whether harassment or abusive conduct is acceptable. It is not.
